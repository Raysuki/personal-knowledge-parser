# ProfileFlow 中文说明

[English README](./README.md)

ProfileFlow 是一款面向学生与求职人群的个人资料智能管理与自动填表工具。系统支持用户上传成绩单、获奖证书、简历、实习证明、科研成果等多类型资料，通过 OCR、文档解析与智能抽取技术，构建可持续更新的个人动态知识库。

基于该知识库，系统能够自动识别网页表单或文档模板中的字段，并将最匹配的个人信息一键填充到对应位置。产品同时提供人工校对、版本追踪、冲突处理与多知识库切换能力，帮助用户高效完成求职、升学、比赛报名等高频申请任务。

## 功能特点

- 支持上传图片、PDF、DOCX、TXT，或直接粘贴文本
- 通过 OCR 与大模型抽取结构化个人资料
- 支持手动核对、补充与修正识别结果
- 支持多知识库管理，适配不同申请场景
- 支持 Word 模板自动填充与导出
- 适用于求职、保研、升学、竞赛报名等重复填表场景

## 最新更新

- 修复了部分环境下“上传图片后识别结果为空白”的问题
- 兼容新版与旧版 PaddleOCR 的不同返回结构，避免识别到文字却被错误过滤

## 技术栈

- 前端：Next.js 15、React 19、TypeScript、Tailwind CSS
- 后端：FastAPI、Python
- 解析能力：PaddleOCR、PyMuPDF、python-docx、Ark runtime SDK
- 存储：SQLite

## 操作说明

### 1. 功能概览

ProfileFlow 主要用于管理个人申请材料，并将材料中的关键信息自动整理为可持续更新的资料库。左侧悬浮工具栏通常包含以下入口：

- 新建资料库：为不同申请场景建立独立资料空间
- 解析资料：上传文件或粘贴文本，让系统自动抽取信息
- 一键填表：将资料库中的信息自动填入指定模板

### 2. 建立并管理资料库

步骤 1：点击左侧工具栏中的“新建资料库”。

建议根据用途分别建立资料库，例如“秋招求职”“保研申请”“竞赛报名”等，便于后续分类管理和信息调用。

步骤 2：输入资料库名称并完成创建。

同一用户可能拥有多套不同版本的材料，分库管理可以有效避免信息混用。

步骤 3：进入详情页后，单击资料库名称即可编辑或重命名。

如果后续申请方向发生变化，可以通过重命名或新建资料库的方式继续维护。

### 3. 解析资料并更新内容

步骤 1：先选中目标资料库，再点击“解析资料”。

请确认当前选中的资料库与用途一致，避免把求职材料误存到其他资料库中。

步骤 2：上传待解析材料，或直接粘贴文本内容。

系统支持图片、PDF、Word 文档和纯文本。建议优先上传清晰、完整、信息较集中的文件，例如正式简历、成绩单、获奖证书、项目证明等。

步骤 3：等待系统自动识别并生成结构化信息。

识别完成后，系统会将姓名、联系方式、教育经历、获奖经历、项目经历等内容展示在右侧详情区域。

步骤 4：人工核对并补充内容。

若识别结果与原材料不一致，请直接在详情页中修改；若某些字段未识别到，也可以手动补充。

步骤 5：点击右上角“保存入库”。

只有保存后，解析结果才会正式写入当前资料库，后续才能用于一键填表。

### 4. 一键填表

步骤 1：选中需要调用的资料库。

请先确认资料库中的信息已经核对完成，并且包含本次填表所需的大部分内容。

步骤 2：点击“一键填表”，上传需要填写的 Word 模板。

模板建议使用结构较清晰、字段较明确的 Word 文档，这样系统更容易完成准确匹配。

步骤 3：等待系统自动填充并下载文档。

文档生成后会自动下载到本地。下载完成后，建议再次检查姓名、联系方式、时间、奖项等级、绩点排名等关键信息。

### 5. 使用建议

- 上传图片时，尽量使用清晰扫描件或高质量截图，避免模糊、反光、裁切不完整等情况影响识别效果
- 若同一信息存在多个版本，例如不同阶段的简历或成绩单，建议分别建库，或在保存前仔细核对
- 若识别内容为空或字段缺失，可以尝试更换更清晰的文件，或直接补充文本后再次解析
- 一键填表适合高频、重复性申请场景，但在正式提交前仍建议进行人工复核

### 6. 推荐使用顺序

第一次使用时，推荐按照以下顺序完成操作：

1. 新建资料库
2. 上传并解析资料
3. 人工核对
4. 保存入库
5. 上传模板
6. 一键填表

这样可以最大程度保证资料库完整、字段匹配准确、输出结果可直接使用。
